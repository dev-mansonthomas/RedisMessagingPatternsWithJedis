🧠 Pseudo-code / logic

XADD chat:companyId:userId * role user content "…" msgId ulid()
1 user sends a message
👤 UI → POST /message(text)
function  postMessage(cid, text):
ensureConversation(cid)
// XGROUP CREATE (idempotent)
XADD chat:cid role=user
// group before XADD → no miss
2 responder generates (1 VT/cid)
loop:
id,msg = XREADGROUP cg:responder
if role != user: XACK; skip
ctx = XREVRANGE chat:cid ..id
llm.generate(ctx):
onToken: XADD chat:cid:tok
onComplete: XADD assistant
EXPIRE tok 60; XACK
3 tokens stream to the user
loop:
es = XREAD chat:cid:tok lastId
for e in es:
ws.push(cid, TOKEN, e.token)
👤 UI renders token
1

The user posts a message. The browser POSTs it; ensureConversation creates the stream and the cg:responder group if absent (idempotent, MKSTREAM) and lazily starts the per-conversation workers, then XADD chat:cid appends the turn — the conversation is the stream (ordered, replayable).

2

The responder generates the reply. One Virtual Thread blocks on XREADGROUP cg:responder (it wakes on a new message — no polling), skips non-user turns, rebuilds the context with XREVRANGE, calls the LLM, XADDs each generated token to chat:cid:tok, then appends the finished assistant turn and XACKs the user message.

3

Tokens stream back to the user. A second Virtual Thread XREADs the token stream and pushes each token over WebSocket (only to this cid's subscribers); the UI appends them, so the answer appears word by word, at the pace the LLM generates it.
